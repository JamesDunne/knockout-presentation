KnockoutJS

A view model data binding library

What is Knockout?

“Knockout is a JavaScript library that helps you to create rich, responsive display and editor user interfaces with a clean underlying data model. Any time you have sections of UI that update dynamically [...], KO can help you implement it more simply and maintainably.”

Why Knockout?

  • Declarative bindings
  • Automatic UI refresh
  • Dependency tracking
  • Templating
  • Very low learning curve

Declarative bindings: Easily associate DOM elements with model data using a concise, readable syntax

Automatic UI refresh: When your data model's state changes, your UI updates automatically

Dependency tracking: Implicitly set up chains of relationships between model data, to transform and combine it

Templating: Quickly generate sophisticated, nested UIs as a function of your model data

Knockout features

  • Separation of HTML presentation from JavaScript logic
  • Ideal for collaborative environment and division of labor
  • UI gets updated automatically when view model changes and vice versa

Why not just use jQuery?

  • Knockout and jQuery solve different problems.
  • Both can co-exist well together.
  • If you do use both, just use jQuery for animations, effects, and convenient components.

Ok, so...

Why use Knockout?

  • jQuery UI manipulation is low-level and repetitive per binding.
  • Knockout binds the whole UI at a high level, binding all elements to a single view model.
  • jQuery UI binding is explicit by CSS selectors ('#id', '.class').
  • Knockout binds UI via data-bind attributes in HTML in a single call.
  • Knockout bindings are easily copyable to multiple elements.
  • Knockout state is held in the view model not DOM elements; jQuery state is held in ... ?

Why use Knockout?

  • Knockout's observable pattern is a game-changer for UI development.

The View is...


<span data-bind="text: message"></span>

<select data-bind="options: tickets,
                   optionsCaption: 'Choose...',
                   optionsText: 'name',
                   value: chosenTicket"></select>

<button data-bind="click: runMe">Run!</button>

just HTML code with data-bind attributes applied to any element!

The View Model is...


function ViewModel() {
  this.message = ko.observable("Hello!");
  this.counter = ko.observable(1);
}

just a JavaScript object with observable properties!

The Model is...

View Model Binding


var vm = new ViewModel();
// Apply bindings to the whole document:
ko.applyBindings(vm);
// or only apply bindings to a specific element and its children:
ko.applyBindings(vm, document.getElementById("onlyThisElement"));

Instantiate the view model object and call 

ko.applyBindings()
to apply the bindings in the HTML to the view model.

Observables

  • ko.observable()
    creates an observable value.
  • Subscribe to it to receive notifications when its value changes.
  • Update its value to have subscribers notified.

Computed Observables

  • ko.computed(function() {})
    creates a computed observable.
  • Automatically subscribes to other observables during evaluation.
  • Evaluates when any subscriptions change.

function ViewModel() {
  this.a = ko.observable(1);

  // Computed observable `b` subscribes to `a`, gets its value
  // as a number and adds 1.
  this.b = ko.computed(function() {
    return Number(this.a()) + 1;
  }, this);
}
